# Analyze Existing Project

扫描已有项目目录，从故事原文和图片文件中提取信息，生成 `elements_config.json` 和其他支持后续故事板生成的配置文件。

## 使用场景

- 已有剧集的原文和参考图，需要继续创作后续集数
- 需要梳理现有项目的角色、场景、物品信息
- 从零散的文件中建立结构化的项目配置

## 输入要求

1. **项目目录路径**（必需）
   - 包含故事原文和/或参考图的目录

2. **目录结构**（灵活）
   - 可以有子目录结构，也可以是扁平结构
   - 支持任意目录组织方式

3. **文件命名规范**
   - **故事文本**：`.txt` 文件，包含剧情、对话、角色描述
   - **角色图片**：文件名包含角色名和变体信息，如 `林小满_日常.png`, `林小满_开心.png`
   - **场景图片**：文件名包含场景名和变体信息，如 `蛋糕店_白天.png`, `蛋糕店_夜晚.png`
   - **物品图片**：文件名包含物品名和变体信息，如 `蛋糕_草莓.png`, `蛋糕_巧克力.png`

## 工作流程

### 1. 扫描项目目录

使用 Glob 工具扫描目录：
- 查找所有 `.txt` 文件（故事原文）
- 查找所有图片文件（`.png`, `.jpg`, `.jpeg`, `.webp`）
- 记录文件路径和文件名

### 2. 读取并分析故事原文

读取所有 `.txt` 文件：
- 理解故事情节、时代背景、风格
- 提取角色信息（名字、年龄、性格、外貌、服装）
- 提取场景信息（地点、时间、氛围、环境描述）
- 提取物品信息（重要道具、物件）
- 记录故事时长

### 3. 分析图片文件

对每个图片文件：

**步骤 1：解析文件名**
- 文件名格式：`名称_变体.扩展名`
- 示例：`林小满_开心.png` → 名称"林小满"，变体"开心"
- 如果没有下划线，整个文件名作为名称，变体为"default"

**步骤 2：读取图片内容**
- 使用 Read 工具读取图片（Read 支持图片文件）
- 分析图片内容：是人物、场景还是物品？
- 描述图片的视觉特征（外貌、服装、环境、物体等）

**步骤 3：结合原文判断分类**
- 将文件名中的"名称"与故事原文对比
- 如果名称在故事中是角色名 → 分类为 character
- 如果名称在故事中是场景/地点 → 分类为 scene
- 如果名称在故事中是物品/道具 → 分类为 prop
- 如果无法判断，根据图片内容智能推断：
  - 图片主体是人物 → character
  - 图片是环境/场景 → scene
  - 图片是物体/道具 → prop

**步骤 4：生成 AI prompt**
- 基于图片内容生成描述性 prompt
- 角色：描述外貌、服装、姿态、风格
- 场景：描述环境、氛围、时间、风格
- 物品：描述物体特征、细节、风格
- 确保 prompt 符合故事的时代背景和视觉风格

### 4. 生成 elements_config.json

整合所有分析结果，生成配置文件：

**角色配置**：
- `name`: 从文件名或故事中提取
- `age`: 从故事中提取（如果有）
- `personality`: 从故事中提取角色性格描述
- `variants`: 每个图片对应一个变体
  - `name`: 变体名称（从文件名提取）
  - `ai_prompt`: 基于图片内容生成的描述性prompt
  - `reference_image`: 相对路径

**场景配置**：
- `name`: 从文件名或故事中提取
- `description`: 从故事中提取场景描述
- `variants`: 每个图片对应一个变体
  - `name`: 变体名称
  - `ai_prompt`: 基于图片内容生成
  - `reference_image`: 相对路径

**物品配置**：
- `name`: 从文件名或故事中提取
- `description`: 从故事中提取物品描述
- `variants`: 每个图片对应一个变体
  - `name`: 变体名称
  - `ai_prompt`: 基于图片内容生成
  - `reference_image`: 相对路径

**AI prompt 生成原则**：
- 准确描述图片中的视觉内容
- 符合故事的时代背景（避免穿帮）
- 符合故事的视觉风格
- 简洁明确，便于图片生成

### 5. 生成项目报告

创建 `project_analysis.md`，包含：

**项目概览**：
- 剧名（从故事或目录名推断）
- 故事梗概（简要总结）
- 故事时长
- 时代背景和视觉风格

**元素统计**：
- 角色数量及列表
- 场景数量及列表
- 物品数量及列表
- 每个元素的变体数量

**详细清单**：
- 每个角色的详细信息（年龄、性格、变体）
- 每个场景的详细信息（描述、变体）
- 每个物品的详细信息（描述、变体）

**质量检查**：
- 图片是否符合时代背景
- 图片是否符合故事描述
- 是否有文字气泡需要移除
- 风格是否一致

**缺失信息**：
- 故事中提到但没有参考图的元素
- 需要补充的描述信息
- 需要用户确认的分类

**后续建议**：
- 如何使用生成的配置文件
- 是否需要生成额外的参考图
- 下一步创作建议

### 6. 处理特殊情况

**无故事原文**：
- 仅基于图片内容和文件名生成配置
- 在报告中标注需要补充故事信息

**无法分类的图片**：
- 列在报告的"需要确认"部分
- 询问用户该图片的类型和用途

**目录结构混乱**：
- 建议重组为标准结构
- 提供文件移动建议（但不自动执行）

**图片质量问题**：
- 标注有文字气泡的图片
- 标注风格不一致的图片
- 标注时代穿帮的图片

## 输出文件

1. **elements_config.json** - 元素配置文件，可直接用于 `generate-storyboard` 技能
2. **project_analysis.md** - 详细的项目分析报告

## 工作示例

```
用户：分析 D:\projects\my-drama\ 目录

助手：
1. [使用 Glob 扫描目录]
   找到 1 个文本文件：story.txt
   找到 8 个图片文件

2. [读取 story.txt]
   故事是关于林小满经营蛋糕店的温馨故事
   时代背景：现代
   主要角色：林小满（8岁女孩）
   主要场景：蛋糕店

3. [逐个读取并分析图片]
   - 林小满_日常.png：8岁女孩，日常服装，全身像
   - 林小满_开心.png：同一女孩，开心表情
   - 蛋糕店_白天.png：温馨的蛋糕店内景，白天光线
   - 蛋糕_草莓.png：草莓蛋糕特写
   ...

4. [生成 elements_config.json]
   - 1 个角色（林小满），2 个变体
   - 1 个场景（蛋糕店），1 个变体
   - 1 个物品（蛋糕），1 个变体

5. [生成 project_analysis.md]
   包含完整的项目分析和后续建议
```

## 注意事项

1. **智能分析优先**：
   - 不依赖硬编码的关键词
   - 结合文件名、图片内容、故事原文综合判断
   - 充分利用 AI 的理解能力

2. **图片路径**：
   - 配置文件中使用相对路径
   - 相对于项目根目录
   - 便于项目迁移

3. **AI prompt 质量**：
   - 基于实际图片内容生成
   - 不使用模板化的固定格式
   - 确保符合故事的时代背景和风格

4. **时代一致性**：
   - 检查图片中的服装、道具是否符合时代
   - 标注可能的穿帮问题
   - 在报告中提醒用户

5. **风格一致性**：
   - 检查所有图片的视觉风格是否统一
   - 标注风格不一致的图片
   - 建议统一风格的方法

6. **用户确认**：
   - 对于无法确定的分类，主动询问用户
   - 对于质量问题，在报告中明确指出
   - 不做假设，保持透明

7. **灵活处理**：
   - 支持任意目录结构
   - 支持不同的文件命名方式
   - 适应不同的项目组织方式

## 与其他技能的配合

- 生成 `elements_config.json` 后，可以使用 `generate-storyboard` 技能继续创作
- 如果需要生成缺失的参考图，可以调用图片生成接口
